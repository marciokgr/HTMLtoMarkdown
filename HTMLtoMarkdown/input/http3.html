O que é HTTP/3?

HTTP/3 é uma nova versão do HTTP. O protocolo HTTP/3 é suportado pela maioria dos navegadores e servidores modernos. Esta atualização deve trazer benefícios de desempenho principalmente para usuários móveis ou conexões não confiáveis. A ideia principal é substituir o TCP por um novo protocolo, o QUIC, que remove alguns problemas de TCP para HTTP. Assim, o QUIC tem as seguintes vantagens (lista não exaustiva):

·        Configuração de conexão mais rápida combinando handshakes TCP e TLS

·        Menos bloqueio meio de uma melhor lógica de recuperação de perda de pacotes

·        Migração de conexão, para que você não precise se reconectar (handshake) ao se mover entre redes (por exemplo, WIFI para celular)

A partir do .NET 6 existe suporte a HTTP/3 para clientes (incluindo gRPC e servidores). Esta implementação baseia-se em HttpClientKestre l MsQuic, é uma implementação da Microsoft do IETF QUIC protocolo. No momento, o .NET oferece suporte a HTTP/3 em:

·        Windows 11 e Windows Server 2022

·        Linux (você pode precisar instalar usando msquicapt install libmsquic)

Caso utilize o .NET 6, talvez esta funcionalidade ainda esteja em preview, portanto, você precisa habilitá-lo explicitamente no csproj ou no código (EnablePreviewFeatures).

<PropertyGroup>
    <TargetFramework>net6.0</TargetFramework>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
    <EnablePreviewFeatures>true</EnablePreviewFeatures>
  </PropertyGroup>

Criando API usando Servidor (Kestrel)

Para criar uma API que tenha suporte ao HTTP/3 você pode configurar o Kestrel para escutar em HTTP/1, HTTP/2 e HTTP/3. É importante oferecer suporte a protocolos mais antigos, pois nem todos os clientes oferecem suporte a protocolos mais recentes. Além disso, HTTP/3 requer conexões seguras, então você deve usar o .UseHttps

//Configurando HTTP3builder.WebHost.ConfigureKestrel((context, options) =>
{
    options.ListenAnyIP(5001, listenOptions =>
    {
        listenOptions.Protocols = HttpProtocols.Http1AndHttp2AndHttp3;
        listenOptions.UseHttps();
    });
});

E será necessário dar o using, caso seu editor já não o faça sozinho

using Microsoft.AspNetCore.Server.Kestrel.Core;

Você também pode usar o log do W3C e verificar a versão do protocolo usada pelo cliente:

{
  "DetailedErrors": true,
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning",
      "Microsoft.AspNetCore.Hosting.Diagnostics": "Information"
    }
  },
  "AllowedHosts": "*"
}

Ativando log na API W3C na API:

app.UseW3CLogging();

Após subir a API e acessar a API você vai ver o protocolo que está sendo consumindo

Criando um Client (HttpClient) C#

HTTP/3 tem suporte no .NET 6 e .NET 7. Então, você precisa habilitar manualmente o recurso antes de usar HTTP/3 em: HttpClient

·        Editar a opção para adicionar um tempo de execução csproj (System.Net.SocketsHttpHandler.Http3Support)

<Project Sdk="Microsoft.NET.Sdk">
    <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net7.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
    </PropertyGroup>
    <ItemGroup>
           <RuntimeHostConfigurationOption Include="System.Net.SocketsHttpHandler.Http3Support"
            Value="true" />
      </ItemGroup>
  </Project>

Defina a seguinte opção antes de criar a primeira HttpClient

System.AppContext.SetSwitch("System.Net.SocketsHttpHandler.Http3Support", true);

Segue exemplo de código do Client

using System.Net;

System.AppContext.SetSwitch("System.Net.SocketsHttpHandler.Http3Support", true);
try
{
    using var client = new HttpClient()
    {
        DefaultRequestVersion = HttpVersion.Version30,
        DefaultVersionPolicy = HttpVersionPolicy.RequestVersionExact
    };
    using var request = new HttpRequestMessage(HttpMethod.Get, "https://localhost:5001/");
    request.Version = HttpVersion.Version30;
    request.VersionPolicy = HttpVersionPolicy.RequestVersionExact;
    using var response = await client.SendAsync(request);
    var data = await response.Content.ReadAsStringAsync();
}
catch(Exception e)
{
    Console.WriteLine(e.Message);
}

Criando um Client JavaScript

Lembrando que seu Windows e navegador tem que ser compatível.

fetch('https://localhost:5001/', {
    method: 'GET',
    headers: {
        'Accept': 'application/json',
        'Content-Type': 'application/json'
    },
    protocol: 'http3',
})
    .then(response => {
        if (!response.ok) {
            throw new Error('Erro na internet');
        }
        return response.json();
    })
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error('Ocorreu um erro:', error);
    });

Vantagens?
